## 1. نموذج العمل

**إنڤير وسيط وعميل** — تشتري وتبيع المواد القابلة لإعادة التدوير.

```
البائع  ──بيع مواد──▶  إنڤير  ──بيع مواد──▶  المشتري
           ◀──رصيد──              ──▶ فاتورة ضريبية
```

### قاعدة الوساطة

- إنڤير تشتري من البائعين — تُضيف رصيدهم في محافظهم
- إنڤير تبيع للمشترين — تُصدر فاتورة ضريبية باسمها
- عند السحب — البائع يُصدر فاتورته لعنوان إنڤير (لأنه باع لها)
- إنڤير تحتجز عمولتها وضريبتها تلقائيًا في محفظتها المركزية
- **كل فاتورة يجب أن تحتوي على كمية الكربون المتجنَّب لكل مادة**


## 4. نظام الفواتير

> ⚠️ **قاعدة أساسية:** كل صفقة فاتورتان منفصلتان — لا يُجمَّع بين الصفقات أبدًا.

**جدول:** `invoices`

```sql
id              UUID PRIMARY KEY
invoice_number  VARCHAR UNIQUE   -- مثال: TXN-2026-00817
type            ENUM('buyer_invoice','seller_invoice')
deal_id         UUID REFERENCES deals(id)
issued_by       UUID             -- user_id أو 'invare' ثابت
issued_to       UUID             -- user_id أو 'invare' ثابت
amount_subtotal DECIMAL(18,4)
vat_amount      DECIMAL(18,4)
amount_total    DECIMAL(18,4)
total_co2_avoided DECIMAL(18,4) -- إجمالي كغ CO₂ للفاتورة
currency        VARCHAR(3)
status          ENUM('draft','issued','paid','cancelled')
pdf_url         VARCHAR          -- رابط ملف PDF المولّد
created_at      TIMESTAMP WITH TIME ZONE
```

**جدول بنود الفاتورة:** `invoice_items`

```sql
id              UUID PRIMARY KEY
invoice_id      UUID REFERENCES invoices(id)
material_id     UUID REFERENCES materials(id)
material_name   VARCHAR
quantity_kg     DECIMAL(18,4)
unit_price      DECIMAL(18,4)
subtotal        DECIMAL(18,4)
co2_per_kg      DECIMAL(10,4)   -- من جدول carbon_values وقت الصفقة
co2_avoided     DECIMAL(18,4)   -- = quantity_kg × co2_per_kg
```

---

### 4.1 فاتورة إنڤير للمشتري

**المصدِّر:** إنڤير | **المستلم:** المشتري

تُولَّد تلقائيًا عند اتمام صفقة شراء.

**محتوى الفاتورة:**

| الحقل | المصدر |
|-------|--------|
| رقم الفاتورة | تسلسل تلقائي `TXN-YYYY-XXXXX` |
| رقم الصفقة | `deals.id` — رابط مباشر |
| التاريخ | تاريخ اتمام الصفقة |
| البائع | إنڤير + الرقم الضريبي |
| المشتري | اسم المستخدم + رقمه الضريبي |
| المواد / الكميات / الأسعار | `deal_items` — سطر لكل مادة |
| كمية CO₂ المتجنَّب / مادة | `carbon_values.co2_per_kg × quantity` |
| إجمالي CO₂ المتجنَّب | مجموع كل المواد |
| المجموع قبل الضريبة | محسوب |
| ضريبة ق.م (5%) | محسوب |
| الإجمالي المستحق | = المبلغ المخصوم من محفظة المشتري |
| حالة الدفع | مدفوعة (من المحفظة) |

---

### 4.2 فاتورة البائع على إنڤير (نموذج مساعد)

**المصدِّر:** البائع | **المستلم:** إنڤير

> إنڤير هي المشتري — لذلك البائع هو من يصدر الفاتورة.
> النظام يولّد **نموذجًا معبّأً مسبقًا** من بيانات الصفقة الفعلية.
> البائع يراجعه ويضيف بياناته البنكية ويرفعه مع طلب السحب.

**آلية التوليد:**
1. النظام يسحب بيانات الصفقة تلقائيًا (مواد، كميات، أسعار)
2. يحسب CO₂ المتجنَّب من `carbon_values`
3. يعرض النموذج للبائع — الأسعار والكميات **مقفلة** (لا يعدّلها البائع)
4. البائع يملأ: بيانات البنك + رقم الهاتف + الإيميل
5. البائع يوقّع إلكترونيًا ويرفع الفاتورة مع طلب السحب

**محتوى الفاتورة:**

| الحقل | المصدر | قابل للتعديل؟ |
|-------|--------|:---:|
| المصدِّر (البائع) | بيانات المستخدم | ✗ |
| المستلم | إنڤير — ثابت | ✗ |
| رقم الصفقة المرجعي | `deals.id` | ✗ |
| المواد / الكميات / الأسعار | `deal_items` | ✗ |
| كمية CO₂ المتجنَّب / مادة | `carbon_values` | ✗ |
| إجمالي CO₂ المتجنَّب | محسوب | ✗ |
| المجموع | = مبلغ السحب | ✗ |
| اسم البنك | يملأه البائع | ✓ |
| رقم الحساب | يملأه البائع | ✓ |
| IBAN | يملأه البائع | ✓ |
| رقم الهاتف | من الملف أو يملأه | ✓ |
| الإيميل | من الملف أو يملأه | ✓ |

## 5. جدول قيم الكربون

**جدول:** `carbon_values`
يُدار من لوحة الأدمن — كمية CO₂ لكل كيلو لكل نوع مادة.

```sql
id              UUID PRIMARY KEY
material_id     UUID REFERENCES materials(id)
material_name   VARCHAR
co2_per_kg      DECIMAL(10,4)   -- كغ CO₂ متجنَّب لكل كغ مادة
source          VARCHAR         -- مرجع المعامل (IPCC / دراسة محلية...)
effective_from  DATE
updated_by      UUID REFERENCES admins(id)
updated_at      TIMESTAMP WITH TIME ZONE
```

**قيم معيارية مبدئية (IPCC 2021):**

| المادة | كغ CO₂ / كغ |
|--------|:-----------:|
| بلاستيك PET | 0.70 |
| ألومنيوم | 9.50 |
| ورق مقوى | 1.15 |
| زجاج | 0.30 |
| حديد / معادن | 1.80 |
| نحاس | 3.50 |

> ⚠️ القيمة المستخدمة في الفاتورة هي القيمة **وقت الصفقة** — لا تتأثر بالتحديثات اللاحقة.

## 6. طلبات السحب

### 6.1 جدول طلبات السحب

**جدول:** `withdrawal_requests`

```sql
id                   UUID PRIMARY KEY
user_id              UUID REFERENCES users(id)
deal_id              UUID REFERENCES deals(id) NULL  -- صفقة مرتبطة (اختيارية)
amount               DECIMAL(18,4)
currency             VARCHAR(3)
bank_name            VARCHAR NOT NULL
account_number       VARCHAR NOT NULL   -- مشفّرة في DB
iban                 VARCHAR NOT NULL   -- مشفّرة في DB
account_holder       VARCHAR NOT NULL
phone                VARCHAR NOT NULL
email                VARCHAR NOT NULL
seller_invoice_url   VARCHAR NOT NULL   -- فاتورة البائع — إلزامية
status               ENUM('pending','approved','rejected','processing','completed')
admin_id             UUID REFERENCES admins(id) NULL
admin_note           TEXT NULL
receipt_url          VARCHAR NULL       -- إيصال التحويل البنكي بعد الإتمام
created_at           TIMESTAMP WITH TIME ZONE
processed_at         TIMESTAMP WITH TIME ZONE NULL
```

---

### 6.2 مسار طلب السحب

```
المستخدم يقدّم الطلب
        │
        ▼
يُجمَّد المبلغ من available_balance
        │
        ▼
   الحالة: pending
        │
        ▼
إشعار الأدمن المالي (إيميل + إشعار)
        │
   ┌─────┴─────┐
   ▼         ▼
موافقة      رفض
   │           │
   ▼           ▼
processing  فكّ التجميد
   │        يعود للرصيد
   ▼        إيميل بالسبب
تحويل بنكي
   │
   ▼
completed
يُخصَّم نهائيًا
إصال بنكي مرفق
إيميل + فاتورة للمستخدم
```

---

### 6.3 قواعد السحب

- رفع فاتورة البائع **إلزامي** — لا يُقبَل الطلب بدونها
- الحد الأقصى: **3 طلبات شهريًا** لكل مستخدم
- لا يمكن سحب رصيد مجمَّد
- الحد الأدنى للسحب: يُحدَّد في إعدادات المنصة

## 8. كشف الحساب

### 8.1 كشف حساب المستخدم

صفحة خاصة بكل مستخدم — تعرض:
- جميع حركات المحفظة: إيداع، سحب، شراء، بيع، خدمات، مزادات، دفعات آجلة
- الرصيد بعد كل عملية
- رابط مباشر لكل مصدر (الصفقة / الخدمة / الطلب)
- رابط للفاتورة المرتبطة
- فلترة: النوع، التاريخ، الحالة، المبلغ
- تصدير: PDF أو Excel

### 8.2 كشف حساب محفظة إنڤير (للأدمن)

- جميع العمليات والضرائب الواردة مع مصدر لكل منها
- الإضافات اليدوية مع اسم الأدمن ورابط الإصال
- ملخص دوري: إجمالي الإيرادات / المصروفات / الصافي

### 8.3 كشف حساب أي مستخدم (للأدمن)

- نفس محتوى كشف المستخدم + الحساب المالي
- سجل الإضافات والخصومات اليدوية

## 11. ملاحظات للمطور

### ⚠️ قواعد إلزامية

```
✔  كل المبالغ → DECIMAL(18,4) — ممنوع float أو double
✔  كل التاريخ في DB → UTC — التحويل للمنطقة المحلية في الواجهة فقط
✔  Idempotency Key لكل عملية مالية — منع التكرار
✔  كل تغيير في balance → عبر service layer — لا كتابة مباشرة في DB
✔  بيانات البنك (IBAN، رقم الحساب) → مشفّرة في DB
✔  Rate limiting: 3 طلبات سحب / شهر / مستخدم
✔  Audit log لكل عملية مالية (لا يُحذف)
✔  API المالية → JWT + IP whitelist للأدمن
✔  ملفات الفواتير والإصالات → التحقق من النوع (PDF/JPG/PNG) والحجم ≤ 5MB
✔  فشل أي خطوة في السحب → Rollback كامل (Database Transaction)
✔  قيمة CO₂ في الفاتورة → تؤخذ وقت الصفقة وتُحفَظ — لا تتغيّر بعد ذلك
```

### 🗺️ مخطط العلاقات (ERD مبسّط)

```
users
  └── user_wallets (1:1)
         └── wallet_transactions (1:N)
                └── invoices (N:1)
                       └── invoice_items (1:N)
                              └── carbon_values (N:1)

deals
  └── wallet_transactions
  └── invoices
  └── deal_items
         └── materials
                └── carbon_values

withdrawal_requests
  └── user_wallets
  └── wallet_transactions
  └── invoices (seller_invoice)

admin_wallet_adjustments
  └── user_wallets
  └── admins
```

## 12. نقاط قيد التحديد

| النقطة | الخيارات | الحالة |
|--------|---------|--------|
